# OpenCrew - AI駆動の動画素材自動選択パイプライン

LoLチャンピオン紹介動画（YouTube Shorts）の制作を自動化するAIパイプライン。
台本JSONと素材を用意すれば、Claude Vision APIが素材選択・画像合成・レイアウト確認まで行い、
人間は確認ボタンを押すだけで動画が完成する世界を目指す。

## ゴール

**「台本と素材を置いたら、あとはAIが判断して動画を出力する」**

- 台本JSONからチャンピオン名・場面の雰囲気を自動解析
- 素材フォルダから適切な画像・動画を自動選択
- いらすとやの顔をチャンピオンアイコンに自動差し替え
- レイアウト（改行位置・素材配置）をAIが目視チェック
- 不足素材はデスクトップ通知で人間に知らせる
- 判断データを蓄積し、段階的に完全自動化

## 現在のステータス

**v0.1.0 - 基盤実装完了（確認モード）**

全判断に人間の確認を要求する初期段階。
データが蓄積されたら `mode: "auto"` に切り替えて自動化する。

## アーキテクチャ

```
台本JSON
    ↓
[script_analyzer] チャンピオン名・場面コンテキスト抽出
    ↓
[asset_finder] 素材フォルダからglob検索
    ├── splash/{チャンピオン名}_{番号}.png
    ├── icons/{チャンピオン名}_{番号}.png
    ├── irasutoya/{キーワード}.png
    └── cinematic/*.mp4（手動配置、AI判断で使用）
    ↓
[pipeline] 各セリフに素材を割り当て
    ├── 確認モード → ターミナルで人間が承認
    └── 自動モード → 過去の判断データから自動決定
    ↓
[face_compositor] いらすとや + チャンピオンアイコン顔合成
    └── Claude Vision API で顔位置検出 → Pillow で差し替え
    ↓
[layout_checker] レンダリング結果のレイアウトQA
    └── Claude Vision API でフレーム画像を目視チェック
    ↓
[notifier] 不足素材のデスクトップ通知
    ↓
enriched script JSON（asset フィールド追加済み）
    ↓
既存の generate_video.py → MP4出力
```

```
opencrew_assets/
├── splash/              # スプラッシュアート
│   ├── シェン_0.png      # _0 = デフォルト → 背景に使用
│   ├── シェン_1.png      # _1 = 別スキン → コンテンツエリアに配置
│   ├── シェン_2.png      # _2 = 別スキン
│   ├── ゼド_0.png
│   └── ゼド_1.png
│
├── icons/               # チャンピオンアイコン（いらすとや顔合成用）
│   ├── シェン_0.png      # _0 = デフォルトアイコン
│   ├── シェン_1.png      # _1 = 別スキンアイコン
│   ├── ゼド_0.png
│   └── ゼド_1.png
│
├── irasutoya/           # いらすとや素材
│   ├── 戦い.png          # 戦闘シーン用
│   ├── 友達.png          # 友情シーン用
│   ├── 裏切り.png        # 裏切りシーン用
│   └── 泣く.png          # 悲しいシーン用
│
└── cinematic/           # シネマティック動画
    ├── awakening.mp4     # ※チャンピオン名はファイル名に含まない
    ├── warrior.mp4       # ※手動で関連動画を配置
    └── rise.mp4          # ※AIが内容から使える箇所を判断
```

### 素材の命名規則

| フォルダ | 命名規則 | 説明 |
|---------|---------|------|
| `splash/` | `{チャンピオン名}_{番号}.png` | `_0`=デフォルト背景, `_1`以上=別スキン（コンテンツエリア配置） |
| `icons/` | `{チャンピオン名}_{番号}.png` | `_0`=デフォルト, `_1`以上=別スキン |
| `irasutoya/` | `{キーワード}.png` | 場面の雰囲気に応じたキーワード |
| `cinematic/` | 任意 | チャンピオン名不要。手動配置、AIが内容を解析して使用 |

## 使い方

### 基本

```bash
python generate_video.py script.json -o output.mp4 --opencrew
```

### 動画レイアウト（YouTube Shorts: 1080x1920 縦長）

```
┌──────────────────────────┐
│   【タイトル帯】          │  ← 上部固定
│   メインタイトル          │
│   サブタイトル            │
├──────────────────────────┤
│                           │
│   【コンテンツエリア】     │  ← 別スキンスプラッシュ /
│   動画 / 合成画像         │     いらすとや合成 /
│                           │     シネマティック動画
│                           │
├──────────────────────────┤
│   【字幕帯】              │  ← 下部固定
│   セリフテキスト          │
└──────────────────────────┘

※ 最背面: デフォルトスプラッシュアート（_0）が全面を覆う
```

### 3つの素材パターン

| パターン | 使い所 | 素材 |
|---------|--------|------|
| **スプラッシュのみ** | 導入・締め・厳かな場面 | `_0`が背景全面。`_1`以上はコンテンツエリア |
| **いらすとや合成** | 感情表現・日常的な場面 | いらすとやキャラの顔をチャンピオンアイコンに差し替え |
| **シネマティック動画** | 戦闘・ドラマチックな場面 | ロゴ等が映らないようAIがクリッピング |

## 設定（config.yaml）

```yaml
opencrew:
  enabled: false                          # true で --opencrew なしでも有効
  mode: "confirmation"                    # "confirmation" / "auto"
  auto_confidence_threshold: 3            # auto で確認スキップする最低確認回数

  asset_dirs:
    splash: "./opencrew_assets/splash"
    cinematic: "./opencrew_assets/cinematic"
    icons: "./opencrew_assets/icons"
    irasutoya: "./opencrew_assets/irasutoya"

  notifications: true                     # デスクトップ通知
  layout_qa: false                        # Vision API レイアウトQA（API コスト注意）
  db_path: "./opencrew_decisions.db"      # 判断履歴DB
```

## 自動化の2段階

### Stage 1: 確認モード（現在）

```
mode: "confirmation"
```

- 全ての素材選択でターミナルに確認プロンプトが表示される
- 「この素材でOK？ [Y/n]」→ 承認するとSQLiteに記録
- いらすとや合成のプレビュー画像も確認可能
- **まずはこのモードでデータを貯める**

### Stage 2: 自動モード（データ蓄積後）

```
mode: "auto"
auto_confidence_threshold: 3
```

- 同じ場面パターンが3回以上確認済みなら自動承認
- 未知のパターンだけ確認を求める
- 蓄積データが増えるほど人間の介入が減っていく

## モジュール構成

```
opencrew/
├── __init__.py           # パッケージ定義
├── pipeline.py           # 全体オーケストレーター
├── script_analyzer.py    # 台本解析（チャンピオン名・場面コンテキスト抽出）
├── asset_finder.py       # 素材検索（番号付き命名規則対応）
├── face_compositor.py    # 顔合成（Vision API顔検出 + Pillow合成）
├── layout_checker.py     # レイアウトQA（Vision APIフレーム確認）
├── decision_store.py     # SQLite判断履歴ストア
└── notifier.py           # デスクトップ通知 + ターミナル確認
```

## 技術スタック

| コンポーネント | 技術 | 用途 |
|-------------|------|------|
| 台本解析 | キーワードマッチ + Claude Haiku | チャンピオン名抽出、場面判定 |
| 顔検出 | Claude Vision API (Haiku) | いらすとや画像の顔バウンディングボックス |
| 画像合成 | Pillow | アイコンの丸切り抜き + 枠線 + 合成 |
| レイアウトQA | Claude Vision API (Haiku) | フレーム画像の品質チェック |
| 判断記録 | SQLite | 確認済み判断の蓄積・検索 |
| デスクトップ通知 | plyer | 素材不足の通知 |
| 動画生成 | VOICEVOX + Pillow + FFmpeg | 既存パイプライン |

## APIコスト

| 処理 | モデル | コスト目安 |
|------|--------|-----------|
| 台本の場面解析 | Claude Haiku | 約$0.001/台本 |
| 顔位置検出 | Claude Haiku | 約$0.005/画像（キャッシュ後は$0） |
| レイアウトQA | Claude Haiku | 約$0.005/フレーム（オプション） |

顔位置はSQLiteにキャッシュされるため、同じいらすとや素材の2回目以降はAPI呼び出し不要。
